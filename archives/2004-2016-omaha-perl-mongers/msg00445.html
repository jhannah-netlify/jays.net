That's not valid barebones XML.  XML only declares 5 internal
entities.  http://www.xml.com/pub/a/98/08/xmlqna1.html#INTENT

HTML on the other hand, declares many:
http://www.w3.org/TR/REC-html40/sgml/entities.html

&oum; could be declared as an eternal entity.. but lets not go there.

I'm guessing &oum; is a typo for the HTML &ouml; which in XML would be
